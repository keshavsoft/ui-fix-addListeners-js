NPM Publish Actions

Details of the publishing workflows (`publish-and-notify.yml` and `npm-publish.yml`).

Workflow Triggers

These actions can be triggered in two ways:

  • Automatic Trigger: Runs whenever a new release is published in the GitHub repository.
  • Manual Trigger: Can be dispatched manually via the GitHub Actions tab by clicking the Run workflow button.

Duplicate Version Safety Check

To prevent build failures when attempting to publish a version that already exists on NPM, we added a validation script. The workflow queries NPM's registry via `npm view` before attempting `npm publish`. If the version matches, publication is skipped, and the job exits successfully:

yaml
- run: | PACKAGE_NAME=$(node -p "require('./package.json').name") PACKAGE_VERSION=$(node -p "require('./package.json').version") if npm view "${PACKAGE_NAME}@${PACKAGE_VERSION}" version >/dev/null 2>&1; then echo "Version ${PACKAGE_VERSION} of ${PACKAGE_NAME} is already published on npm. Skipping publish." else npm publish fi env: NODE_AUTH_TOKEN: ${{secrets.NPM_TOKEN}}

Downstream Workflow Call

Immediately after a successful publish (or when publish is safely skipped), the workflow invokes the downstream reusable workflow `notify-dependents.yml` to propagate the update:

yaml
notify-dependents: needs: publish-npm uses: ./.github/workflows/notify-dependents.yml secrets: REPO_DISPATCH_TOKEN: ${{ secrets.REPO_DISPATCH_TOKEN }}

Update Dependency Action

Details of the automatic update handler workflow (`update-dependency.yml`).

Workflow Trigger

This workflow is triggered when it receives a `repository_dispatch` webhook (specifically, the event type `dependency-updated`) sent from upstream repositories (like `express-fix-any-js`).

yaml
on: repository_dispatch: types: [dependency-updated] workflow_dispatch:

Step 1: Pull and Install Latest Dependency

The runner checks out the branch using the repository dispatch token (which allows write-backs), and executes npm commands to fetch the latest upstream package version:

yaml
- run: npm install - run: npm install express-fix-any-js@latest

Step 2: Automated Version Bump

If package files are altered by the dependency update, a version patch command is triggered to bump the patch version without creating full git tags:

yaml
- name: Bump version if updated run: | git diff --quiet package.json || npm version patch --no-git-tag-version

Step 3: Commit and Push

The new version and lock file modifications are committed using git bot details and pushed back to the main branch:

yaml
- name: Commit and push changes run: | git config --global user.name "github-actions[bot]" git config --global user.email "github-actions[bot]@users.noreply.github.com" git add package.json package-lock.json if ! git diff --cached --quiet; then git commit -m "Update express-fix-any-js to latest and bump version" git push else echo "No changes to commit" fi

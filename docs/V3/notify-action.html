Notify Dependents Action

Details of the notification reusable workflow (`notify-dependents.yml`).

Overview

When a new package version is published, downstream dependents need to be updated. Since they reside in separate repositories, we trigger their workflows via GitHub's **Repository Dispatch API**.

The `notify-dependents` action is modularized as a reusable workflow (`workflow_call`).

Step 1: Registry Replication Pause

NPM metadata queries can take a few moments to replicate globally across NPM CDNs. To prevent downstream jobs from failing to locate the updated package, the action sleeps for **15 seconds** before making the API request.

yaml
- name: Wait for NPM registry replication run: sleep 15

Step 2: API Repository Dispatch

The workflow fires a POST request to the downstream repository dispatches endpoint for `kschema-fs-api-gen-get-actions`, triggering their dependency updater workflow using the `dependency-updated` event type:

bash
curl -X POST \ -H "Accept: application/vnd.github+json" \ -H "Authorization: Bearer ${{ secrets.REPO_DISPATCH_TOKEN }}" \ https://api.github.com/repos/keshavsoft/kschema-fs-api-gen-get-actions/dispatches \ -d '{"event_type":"dependency-updated"}'
